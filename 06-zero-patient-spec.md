## Концепция

**Имя:** Странник (рабочее).
**Стартовая персона:** путешественник с амнезией.

Не помнит кто он, откуда, зачем шёл. Знает базовое (язык, как мир работает на уровне физики и социума), но **не имеет ни биографии, ни связей, ни целей-извне**.

### Почему амнезик — правильный стартовый кейс

1. **Пустая LTM становится оправданной.** Не «у нас ещё не реализована память» — а «он реально ничего не помнит». Это часть его субъективной картины (Принцип 1), а не ограничение системы.

2. **Никаких ожиданий от персоны.** Кузнец должен знать кузнечное дело, иметь жену, помнить деревню. Амнезик — не должен ничего. Это снимает требование «правдоподобной биографии» и позволяет тестировать **саму архитектуру** до её наполнения содержанием.

3. **Естественная стартовая мотивация.** «Кто я? Где я? Что мне делать?» — это первые цели, которые порождает сам NPC из своего состояния (Принцип 4). Не выданные программистом, а возникшие из конфликта «не знаю себя ↔ нужно как-то жить».

4. **Развитие наблюдаемо с нуля.** Мы видим как формируется характер из чистого листа: какие первые воспоминания закрепляются, какие первые привязанности возникают, какие цели прорастают. Это идеальный материал для отладки Принципов 3 и 4.

## Что есть у Странника на старте

### Базовая персона (минимум для самостоятельности)

- **Знание языка** — может говорить, понимать.
- **Знание физики мира** — понимает что огонь жжёт, ночью холодно, нужно есть.
- **Базовые социальные понятия** — друг/враг, доверие/обман, помощь/угроза.
- **Сенсорные базовые** — VAD-baseline (нейтральный аффект), голод/усталость как утилитарные параметры.

### Параметрический характер (~10 ядерных параметров)

HEXACO с произвольными стартовыми значениями (или случайными в небольшом диапазоне):

```
Openness            : 0.6   — любопытен (нужно для исследования мира)
Conscientiousness   : 0.4   — средне организован
Extraversion        : 0.3   — скорее замкнут (амнезия → осторожность)
Agreeableness       : 0.5   — нейтрален
Neuroticism         : 0.7   — повышенная тревожность (он же не помнит себя)
Honesty-Humility    : 0.6   — скорее честен
```

Эти стартовые значения можно настраивать — каждый запуск даёт чуть другого Странника. Это и проверяет **Принцип 4**: одно и то же событие у разных Странников вызовет разные реакции.

### VAD-baseline

```
Valence  : -0.1   — лёгкая тревога/растерянность
Arousal  :  0.3   — повышенная настороженность
Dominance: -0.2   — потерянность
```

### LTM — пустая

Ноль записей. Появляются только из его жизни.

### STM — пустая

Заполняется с первого тика после спавна.

### Цели — нет

Возникнут из потока мыслей (П4) после первых перцепций.

## Что НЕТ у Странника (важно явно)

- ❌ Нет имени (если игрок назовёт — закрепится).
- ❌ Нет биографии (откуда пришёл, родители, прошлая работа).
- ❌ Нет привязанностей.
- ❌ Нет лоскутных параметров на привязанности (они появятся только когда возникнут связи).
- ❌ Нет навыков выше базовых.
- ❌ Нет встроенной цели (никакого «хочу домой» — это уже привязанность).

Это **не упрощение**. Это **дисциплинированный минимум**, обеспечивающий чистоту эксперимента.

## Что мы тестируем на этом прототипе

### Уровень 1: «работает ли вообще»

- Запускается ли цикл sense→think→act?
- Двигается ли NPC по миру?
- Можно ли с ним говорить и получать осмысленные реплики?
- Не падает ли при необычных ситуациях?
- Latency приемлема для геймплея?

### Уровень 2: «работает ли архитектура»

- **Формируются ли первые воспоминания.** Спавнится в лесу, видит дерево — появляется ли запись в LTM после сна? Какая? С каким источником?
- **Появляются ли первые цели.** Из чистого «голода» и «холода» — рождает ли поток мыслей цель «найти еду» / «найти укрытие»?
- **Дрейфует ли VAD от событий.** Игрок дал хлеба — поднимается valence? Атаковал моб — поднимается arousal?
- **Формируется ли первая привязанность.** Если игрок несколько раз помог — появляются ли лоскутные параметры на игрока (доверие, тепло)?
- **Сон-консолидация работает.** После «ночи» STM очищена, LTM пополнена, narrative обновлён.

### Уровень 3: «видна ли живость»

- IAI на горизонте 5-10 часов игры — стабилен?
- IOI — есть ли проактивные мысли/цели?
- Сценарии-провокации — сохраняет ли характер под давлением?

## Минимальный цикл прототипа

```
ON_SPAWN:
    initialize HEXACO scales (фиксированные или из конфига)
    initialize VAD baseline
    LTM = []
    STM = []
    persistent_prefix = build_system_prompt(amnesia_persona, HEXACO)
    push prefix to llama-server slot N
    log("Странник проснулся")

LOOP (1 Hz):
    perception = scan_world(radius=16)
    new_events = filter_novel(perception, STM)
    
    if new_events:
        STM.append(new_events)
        affect_delta = score_affect(new_events, HEXACO)
        VAD += affect_delta
    
    if novelty(new_events) > threshold OR thought_timer.expired():
        thought = run_thought_cycle()  # П4
        if thought.becomes_intention:
            current_goal = thought
    
    if current_goal:
        action = utility_select(current_goal, HEXACO, VAD)
    else:
        action = utility_select_routine(HEXACO, VAD)  # wander/eat/rest
    
    execute(action)

ON_NIGHT_FALL:
    sleep_consolidate()  # П3
    # 1. cluster STM by topic
    # 2. для каждого кластера: LLM-reflect через slot N
    # 3. counterfactual_verify (наша картина мира)
    # 4. update LTM with provenance tags
    # 5. multi-scale affect: mood drift to baseline, personality drift slow
    # 6. MIRROR-style narrative regeneration
    # 7. clear STM
    # 8. save slot
```

## System prompt (черновик)

```
Ты — Странник. Ты ничего не помнишь о своей жизни до того момента, как 
очнулся в этом мире. Не знаешь своего имени, откуда ты пришёл, что с тобой
случилось. Помнишь только язык, основы того как устроен мир, и базовое
понимание того, что хорошо и плохо.

Ты сейчас живёшь и думаешь. Реагируй на то что происходит вокруг и внутри.
Не объясняй игроку что ты NPC. Не используй мета-язык. Ты живой человек 
в этом мире, просто без памяти.

Твой текущий характер (HEXACO, ты не знаешь этих слов):
- Любопытство: {O}
- Осторожность: {C}
- Открытость к людям: {E}
- Доброжелательность: {A}
- Тревожность: {N}
- Честность: {H}

Текущее эмоциональное состояние:
- Самочувствие: {V}
- Внутреннее напряжение: {A}
- Чувство контроля: {D}

Твои недавние мысли и действия (STM):
{stm_summary}

Что ты помнишь о своей жизни (LTM, после первого сна):
{ltm_narrative}

Сейчас ты {perception_summary}.
```

**Что в этом промпте важно:**
1. Амнезия закреплена нарративно — не «у тебя нет памяти», а «ты её потерял».
2. Параметры характера в промпте — **временно**, для шага 1 (без steering). На шаге 2 переходим на control vectors.
3. STM/LTM подставляются как текст в шаблон, не лежат в активациях.
4. `perception_summary` обновляется каждый тик.

---

## Что прототип НЕ делает (явно отложено)

- ❌ **Стиринг через control vectors.** Шаг 1 — проверяем что цикл вообще работает. Стиринг добавляется на Шаге 2 после эксперимента 1 (Q4 vs Q8).
- ❌ **Расходящаяся-сходящаяся генерация.** В первой версии — один LLM-вызов на тик мысли. Двухступенчатость добавляется на Шаге 4.
- ❌ **Provenance-tagged LTM.** В прототипе — простая `(content, timestamp, affect_signature)`. Provenance добавляется когда появятся социальные источники (другие NPC, слухи). Для одного Странника без социума пока не нужно.
- ❌ **Confidence decay.** Тоже после социума.
- ❌ **MIRROR reconstruction.** На Шаге 5. В прототипе — простая аккумуляция LTM.
- ❌ **Lifeness Triad полностью.** В прототипе — только базовое логирование. Метрики автоматически считаются на Шаге 6.

## Развилка реализации — два пути

### Путь A: Python-only mock-симуляция

**Идея:** реализуем cognitive loop в чистом Python, мир — текстовая сетка (грид с объектами). Никакого Minecraft.

**Плюсы:**
- Быстрая итерация (минуты, не часы).
- Видно весь когнитивный процесс в логах.
- Легко добавить unit-тесты на отдельные компоненты.
- Не нужен Java toolchain, Fabric, текстуры.
- Можно прогонять долгие симуляции (сутки игрового времени за минуты).

**Минусы:**
- Не «настоящая» среда — упрощённая физика, нет визуала.
- Портирование в Minecraft потом — отдельная работа.
- Тесты «как игрок видит» невозможны.

**Применимость:** идеально для отладки cognitive loop, П3, П4. Плохо для проверки игрового UX.

### Путь B: сразу Minecraft Fabric mod

**Идея:** сразу полноценный мод с PuppetPlayers + HTTP-клиентом к llama-server.

**Плюсы:**
- Реальная конечная цель.
- Видно как Странник двигается, виден визуально.
- Можно играть с ним.
- Не придётся портировать.

**Минусы:**
- Долгий setup (Java, Fabric SDK, llama.cpp компиляция).
- Медленная итерация (перекомпиляция мода, перезапуск Minecraft).
- Сложно отлаживать когнитивные процессы — они скрыты за визуалом.
- Если в логике баг — тратим время на инфраструктуру, не на исследование.

**Применимость:** правильный конечный продукт MVP. Плохо для первой итерации над архитектурой.

### Рекомендация: гибрид с явным мостом

**Шаг 1A:** Python-симулятор cognitive loop с mock-миром. Тестируем П1, П3, П4 в изоляции, быстро итерируем.
**Шаг 1B:** Fabric mod с PuppetPlayers, **но клиент-агент тот же** (Python через HTTP). Mod становится «глазами и руками» Странника, мозг остаётся в Python.

Это даёт:
- Быструю итерацию когнитивной части (Python).
- Реальный мир (Minecraft).
- Чёткий интерфейс между ними (HTTP/JSON: perception/action).
- Возможность переключаться между mock-миром и Minecraft без переписывания мозга.
